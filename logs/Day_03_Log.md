## What I Learned

* Array rotation problems help strengthen index movement and loop control.
* Left rotation and right rotation are opposite operations, but both need careful handling of first/last elements.
* Using `k = k % n` is important when rotation count is greater than array size.
* Union of arrays requires combining elements and handling duplicates properly.
* Intersection of sorted arrays can be solved using the two-pointer approach.
* XOR is useful when every number appears twice except one.
* Longest subarray problems require understanding subarray boundaries clearly.
* Frequency counting becomes easier using `map`.
* Percentage increase/decrease problems need clear identification of original value, changed value, and final value.
