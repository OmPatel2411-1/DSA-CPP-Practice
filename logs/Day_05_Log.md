## What I Learned

- Leaders in an array require checking all elements on the right side.
- Longest consecutive sequence becomes easier after sorting the array.
- Duplicate values must be handled carefully in consecutive sequence problems.
- Rearranging by sign becomes easier when positive and negative values are separated first.
- Printing all subarrays helps build a strong foundation for future subarray problems.
- Maximum product subarray is different from maximum sum because negative numbers can change the result.
- Merge two sorted arrays can be solved using the two-pointer technique.
- Duplicate number can be found using sorting and adjacent comparison.
- Missing and repeating number can be found using frequency counting.
- Profit and loss questions require correctly identifying CP, SP, profit, loss, and percentage.

## Mistakes / Errors Faced

- Needed to be careful while checking leaders because only right-side elements matter.
- Longest consecutive sequence required handling duplicates properly.
- Printing all subarrays needed three loops, which can be confusing without dry running.
- Maximum product subarray required attention to negative numbers.
- Merge two sorted arrays required careful pointer movement.
- Missing and repeating number required correct frequency array size.
- Aptitude questions required careful reading of whether CP or SP was given.

## How I Fixed Them

- Dry ran leaders in array from left to right.
- Sorted the array before solving longest consecutive sequence.
- Used separate loops for start, end, and printing subarray elements.
- Used brute force for maximum product to understand the concept first.
- Used two pointers for merging sorted arrays.
- Created a frequency array of size `n + 1` for missing and repeating number.
- Recorded aptitude questions and answers clearly in the log.
