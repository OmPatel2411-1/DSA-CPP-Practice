## What I Learned

- Two Sum can be solved using brute force, but using map improves the approach.
- In Two Sum using map, the key idea is finding the remaining value using `target - current`.
- Sorting 0s, 1s, and 2s can be solved using a counting approach.
- Majority element brute force checks the frequency of every element.
- Maximum subarray sum can be solved using brute force first to understand subarray boundaries.
- Kadane’s Algorithm is an optimized approach for maximum subarray sum.
- Stock buy/sell problem depends on tracking the minimum price so far and maximum profit.
- Rearranging positive and negative numbers becomes easier by separating values into two vectors first.
- Profit and loss aptitude problems require correctly identifying cost price, selling price, profit, and loss.

## Mistakes / Errors Faced

- Needed to understand the difference between brute force and optimized map-based Two Sum.
- Kadane’s Algorithm required dry running because resetting current sum to zero can be confusing at first.
- Stock buy/sell logic needed careful understanding of buying before selling.
- Rearranging positive and negative numbers required handling unequal counts properly.
- Aptitude required attention to whether the given value was cost price or selling price.

## How I Fixed Them

- Solved brute force first before moving to optimized logic.

- Tracked minimum price before calculating stock profit.
